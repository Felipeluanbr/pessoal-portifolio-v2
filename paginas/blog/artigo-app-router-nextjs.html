Guia definitivo para o App Router no Next.js 13

17 de julho de 2023 — 7 min de leitura

O App Router no Next.js é responsável por gerenciar o roteamento do lado do cliente em sua aplicação. Ele fornece uma maneira declarativa e intuitiva de definir rotas, gerenciar navegação e URLs dinâmicas. Aqui estão alguns recursos principais do App Router no Next.js:

  1. Roteamento baseado em arquivo: O Next.js usa um sistema de roteamento baseado em arquivo, onde cada página do seu aplicativo é representada por um arquivo no diretório "pages". O nome do arquivo determina o caminho da URL correspondente. Por exemplo, um arquivo chamado "about.js" no diretório "pages" estará acessível no caminho da URL "/about".
  2. Rotas Dinâmicas: O Next.js permite criar rotas dinâmicas usando colchetes ([ ]) no nome do arquivo ou na estrutura do diretório. Por exemplo, um arquivo chamado "[id].js" pode corresponder a rotas como "/posts/1" ou "/users/john". Os segmentos dinâmicos são acessíveis como parâmetros no componente da página e podem ser usados ​​para buscar dados ou personalizar o conteúdo.
  3. Link Component: Next.js provides the component, which enables client-side navigation between pages. It ensures that the page transition is handled gracefully, without full page reloads. You can use the component to create anchor tags () with client-side routing behavior.
  4. Navegação programática: O Next.js também oferece navegação programática por meio do gancho useRouter. Ele permite que você acesse o objeto roteador, que fornece métodos como push(), replace() e back() para navegar programaticamente dentro do seu aplicativo.
  5. Divisão de código: o Next.js realiza a divisão de código automaticamente, o que significa que apenas o JavaScript e o CSS necessários para a página atual são carregados. Isso ajuda a otimizar o tempo de carregamento inicial da página e melhora o desempenho.
